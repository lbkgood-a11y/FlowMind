## Why

快速开发目前只完成了表单定义与后端应用 API，数据库中的“应用管理”菜单指向不存在的前端组件，导致 404；表单发布、应用编排、用户授权和运行入口没有形成可操作闭环，也缺少列表页快速构建及主—子—孙多表单关系能力。该缺口直接阻断业务人员把表单交付给最终用户使用。

## What Changes

- 修复“快速开发 → 应用管理”404，提供应用草稿、版本、页面、动作、发布和下线管理界面。
- 统一“表单定义 → 发布 → 应用绑定 → 页面/动作编排 → 发布 → 角色授权 → 应用中心运行”的完整流程。
- 增加列表页可视化快速编排，支持列、筛选、排序、展示、行操作与默认布局配置。
- 新增主表、子表、孙表三级表单关系定义，支持一对一/一对多、关联键、级联保存和运行时嵌套编辑/查看。
- 发布时校验关系图无环、引用版本有效、字段映射合法，并固化不可变快照。
- 应用和表单发布后同步语义化授权资源；角色授权后自动投影菜单可见性，应用中心只展示用户可访问应用。
- 增加迁移、服务、前端组件与闭环集成测试。

## Capabilities

### New Capabilities

- `lowcode-form-relations`: 定义主子孙表单关系、关系校验、发布快照、级联数据读写与运行时嵌套呈现。
- `lowcode-list-designer`: 提供列表页字段、筛选、排序、操作和布局的快速编排及运行时渲染契约。

### Modified Capabilities

- `lowcode-form-governance`: 扩展发布治理以包含关系图、关联字段和嵌套表单快照校验。
- `lowcode-application-runtime`: 增加应用管理工作台、页面设计配置、发布闭环和主子孙运行时能力。
- `enterprise-authorization-model`: 明确低代码应用与关联表单发布后的资源同步、角色授权和菜单投影行为。

## Impact

- 后端：`service-lowcode` 数据库迁移、实体、DTO、服务、控制器、运行时与授权同步。
- 权限与菜单：`service-auth` 菜单修复迁移和低代码语义资源授权。
- 前端：web-antd 应用管理、列表设计器、关系设计器、运行时页面及 API 客户端。
- 数据模型新增应用页面设计配置与表单关系表；保持现有单表表单和已发布应用兼容。
- 不引入动态代码执行；所有生成页面继续使用注册组件和结构化 JSON 元数据。
