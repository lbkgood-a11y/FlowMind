## Context

流程包列表已经持有流程包 ID、状态和完整 `processJson`，流程设计器中的 `FlowDesigner` 也具备 JSON 转画布能力，但设计器页面只面向新建场景。组件只在挂载时读取初始 JSON，异步加载不会回显；`readonly` 属性也没有落实到 X6 交互配置。

## Goals / Non-Goals

**Goals:**

- 从流程包列表直接进入已有流程的可视化设计页。
- 草稿可编辑保存，已发布和已下架版本只读查看。
- 保留现有流程包 API、路由和流程 JSON 协议。
- 异步加载后可靠回显画布。

**Non-Goals:**

- 不允许直接修改已发布或已下架流程包。
- 不新增后端接口或流程版本模型。
- 不实现版本差异比较、协同编辑和自动保存。

## Decisions

### 1. 使用查询参数复用设计器路由

列表跳转到 `/process/designer?packageId=<id>`。无 `packageId` 时保持现有新建设计行为，避免增加重复页面。

### 2. 设计器按流程包状态决定模式

加载流程包后，`DRAFT` 进入编辑模式，其他状态进入只读模式。设计器标题显示版本和状态，保存按钮只在草稿且具备更新权限时出现。

### 3. 保留业务配置并回显画布

设计器从 JSON 根节点读取业务对象、流程标识和名称，加载对应业务对象目录后以目录默认值为基线，再用已有定义覆盖可识别配置。画布直接使用原 `processJson` 回显。

### 4. FlowDesigner 响应异步数据并执行只读限制

组件监听 `modelValue` 变化并重新加载画布。只读模式禁用拖拽、连接、删除、键盘修改和属性编辑，但保留缩放、平移与查看。

### 5. 显式加载 X6 插件样式并限制画布比例

Scroller、Selection、Dnd、Snapline 和 MiniMap 的结构依赖各插件 CSS。应用显式声明插件依赖并加载样式，避免插件容器退化为普通文档流导致画布和节点视觉尺寸异常。节点采用紧凑尺寸，加载后使用不超过 100% 的比例适配画布。

### 6. 设计器作为流程包清单的隐藏详情路由

流程包管理是流程定义的唯一清单入口，设计器不再作为平级导航菜单展示。数据库将设计器菜单设为隐藏并绑定流程包查询权限，同时把已拥有流程包菜单的角色补授设计器路由，确保“查看设计图”可达；新增和保存能力仍由 POST/PUT 按钮权限控制。

### 7. 解耦流程包和业务目录加载

流程包详情与 `processJson` 是查看画布的必要数据，业务对象目录只影响业务配置表单和重新保存。两者分别捕获错误：目录加载失败时保留画布只读查看，并明确提示暂不可编辑保存，不再错误显示为流程包不存在。

### 8. 可折叠的设计器工作区

页面级业务配置和流程预览面板各自提供显示/隐藏开关，隐藏后中心画布占用释放的全部空间。画布内部节点库也提供折叠开关；属性面板仍由节点选择触发。布局变化依赖 X6 `autoResize` 自动重新计算画布容器。

### 9. HTML 节点使用确定像素尺寸

X6 HTML 节点位于 `foreignObject` 中，百分比高度在部分布局下会相对画布视口计算。矩形节点根元素直接使用节点定义的像素宽高，圆形节点继续使用固定像素内缩，从源头避免审批节点被拉伸为整页高度。

### 10. 受控画布缩放

画布工具栏提供每次 10% 的缩小和放大、点击比例恢复 100%、适应画布四种操作。缩放范围限制为 25%–200%，并监听 X6 `scale` 事件实时展示当前比例；只读模式同样允许缩放查看。

### 11. 网关路由覆盖业务对象目录

设计器通过网关调用 `/api/v1/process-business-objects`。该接口属于 workflow-engine，但原网关 workflow 路由仅覆盖流程包、实例和任务。将业务对象目录根路径及子路径加入同一路由，保持鉴权、TraceId 和服务边界不变。

### 12. 基于端口的节点连线

HTML 节点注册输入和输出端口：开始节点只有输出端口，结束节点只有输入端口，其余节点同时具备两者。连线只能从输出端口连接到输入端口。节点库同时支持拖拽和双击新增，降低 DnD 在不同浏览器中的操作门槛。

### 13. JSON 与画布双向同步

节点新增、删除、移动、属性修改、连线完成和连线条件变化均触发 JSON 输出。技术预览在草稿模式提供 JSON 编辑器，“应用到画布”校验后更新配置与 `modelValue`，由 FlowDesigner 重新构建画布。

### 14. 已发布流程转草稿设计

已发布和已下架流程保持不可直接修改，但具备版本权限的用户可在设计器中点击“创建新版本并设计”。系统复用版本接口创建草稿，并切换当前设计器的 `packageId` 到新草稿。

### 15. 连线状态与条件表达式引导

新建边在连接完成前不展示条件标签，鼠标释放后若未同时绑定输出和输入端口则立即删除，避免画布残留孤立文本。边数据保存真实条件表达式，标签只负责友好展示。属性面板明确说明支持 `true` 默认分支、字段比较、字符串引号以及 `&&`、`||` 组合条件。

## Risks / Trade-offs

- [历史 JSON 缺少部分业务闭环字段] → 使用业务目录默认值补齐，仅在草稿主动保存时生成完整定义。
- [重新加载 JSON 会清空当前未保存画布] → 仅在外部 `modelValue` 实际变化时加载，页面不在编辑期间重复拉取。
- [已发布流程被前端误改] → 前端隐藏保存并启用只读交互；后端既有状态校验继续作为最终防线。

## Migration Plan

前端发布后入口立即生效，无数据迁移。回滚时移除列表跳转和设计器加载逻辑即可，流程包数据不受影响。
