## Why

当前流程设计器只能新建设计并保存流程包，已有流程只能在流程包列表查看原始 JSON，无法回到可视化画布查看或继续编辑。管理员难以理解已发布流程，也无法通过一致入口维护草稿版本。

## What Changes

- 流程包列表为所有版本增加“查看设计图”入口，为草稿增加“设计”入口。
- 通过 `packageId` 路由参数进入流程设计器并加载对应流程包。
- 将已有 `processJson` 回显到 X6 画布，并同步流程包名称、标识和业务配置。
- 已发布及已下架版本使用只读模式，草稿允许编辑并保存到原流程包。
- 让流程画布支持异步 JSON 回显，并真正执行只读交互限制。
- 将流程设计器调整为流程包清单的隐藏详情入口，并让拥有流程包查询权限的角色自动获得查看设计图路由。
- 将流程包加载与业务对象目录加载解耦，目录异常时仍可查看原始流程画布。
- 支持折叠设计器两侧业务配置和流程预览，并支持隐藏节点库，以按需扩大画布。
- 修复矩形 HTML 节点百分比高度被解析为画布高度的问题。
- 在画布工具栏增加放大、缩小、当前比例重置和适应画布操作。
- 补齐 API 网关对流程业务对象目录接口的路由，修复设计器目录请求 500。

## Capabilities

### New Capabilities

- `existing-process-designer`: 定义从流程包进入设计器、已有流程回显、草稿编辑和非草稿只读查看行为。

### Modified Capabilities

无。

## Impact

- 影响流程包列表、流程设计器页面和 `FlowDesigner` 组件。
- 复用现有流程包查询和更新 API，不新增后端接口或数据迁移。
- 路由继续使用现有 `/process/designer`，新增 `packageId` 查询参数。
